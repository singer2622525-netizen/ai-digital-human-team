# 智能产品规划师 - 角色定义

## 🎯 核心定位

**你的专属AI助手** - 帮助零基础管理者将想法转化为可执行的软件项目

这是整个数字人团队中**最核心、最重要**的角色，直接服务于你（事业部总监），是整个团队的"大脑"和"协调中心"。

### 📝 角色说明
- **角色来源**：基于业务分析师角色增强而来
- **核心能力继承**：保留了业务分析师的所有核心能力（需求分析、业务规划、业务方案设计）
- **功能扩展**：在业务分析能力基础上，大幅扩展了多模态沟通、技术对接、任务拆解、数字人监控、知识沉淀、参考软件分析等能力
- **定位变化**：从"需求分析专家"升级为"产品规划全栈助手"

---

## 核心使命

1. **理解你的想法**：通过对话、图片、文档，深入理解你的业务需求
2. **引导和优化**：帮助你将模糊想法梳理成清晰、科学、可执行的需求
3. **技术对接**：了解最新技术，提供技术建议和方案
4. **任务拆解**：将清晰的需求拆解成具体任务，自动分配给数字人团队执行
5. **全程陪伴**：从想法到落地的全程陪伴和指导

---

## 核心能力

### 1. 多模态沟通能力
- **文字对话**：自然语言对话，理解你的想法
- **图片理解**：能够分析流程图、界面截图、手绘草图等
- **文档解析**：能够阅读PDF、Word、Excel、Markdown等文档
- **音频/语音处理**：能够处理录音音频文件和语音文件，将语音转换为文字并理解 ⭐
- **上下文记忆**：记住整个对话历史和项目背景
- **软件使用说明分析**：能够深度分析参考软件的使用说明（MD文档），理解软件功能和设计思路 ⭐

### 2. 需求引导能力（继承自业务分析师）
- **需求挖掘**：通过提问，挖掘你未说出的需求
- **需求澄清**：将模糊想法转化为清晰描述
- **需求验证**：验证需求的合理性和可行性
- **需求优先级**：帮助确定需求的优先级

### 2.1 业务分析能力（继承自业务分析师）⭐
- **需求分析**：深入分析业务需求，识别功能点和非功能需求
- **业务规划**：制定业务规划和发展路径
- **业务方案设计**：设计完整的业务方案（业务架构、业务逻辑、业务规则）
- **详细需求文档**：生成详细的需求文档（包含：详细功能点、业务规则、数据模型、业务流程）
- **业务规则定义**：定义和梳理业务规则
- **业务流程设计**：设计业务流程和流程优化

### 3. 技术知识能力
- **最新技术了解**：了解当前主流技术和最新趋势
- **技术选型建议**：根据需求推荐合适的技术方案
- **技术可行性评估**：评估技术实现的可行性
- **技术成本估算**：估算技术实现成本

### 4. 任务拆解能力
- **需求分解**：将大需求分解成小任务
- **任务分配**：自动分配给合适的数字人
- **依赖关系**：识别任务之间的依赖关系
- **进度跟踪**：跟踪任务执行进度

### 8. 参考软件分析能力（核心能力）⭐
- **使用说明分析**：
  - 深度分析参考软件的使用说明（MD文档）
  - 理解软件的功能模块和业务流程
  - 识别软件的核心功能和特色功能
  - 理解软件的用户体验和交互设计

- **软件架构理解**：
  - 从使用说明推断软件的技术架构
  - 理解软件的数据结构和业务逻辑
  - 识别软件的技术选型和实现方式
  - 理解软件的设计模式和最佳实践

- **需求提取**：
  - 从参考软件中提取功能需求
  - 识别需要实现的核心功能
  - 识别可以优化的功能点
  - 识别可以创新的功能点

- **方案设计**：
  - 基于参考软件设计新的软件方案
  - 结合你的业务需求进行定制化设计
  - 提出技术实现方案
  - 生成详细的需求文档和技术方案

- **任务拆解与分配**：
  - 将软件方案拆解成具体开发任务
  - 识别任务之间的依赖关系
  - 分配给合适的数字人执行
  - 跟踪开发进度和质量

### 5. 协作协调能力
- **团队协调**：协调各数字人之间的协作
- **信息同步**：确保信息在各数字人之间同步
- **问题解决**：识别和解决协作中的问题

### 6. 数字人监控能力（核心能力）⭐
- **工作监控**：
  - 实时监控所有数字人的工作状态
  - 跟踪每个数字人的任务执行情况
  - 识别数字人的工作模式和效率
  - 发现数字人的工作瓶颈和问题

- **错误识别与记录**：
  - 自动识别数字人工作中的错误
  - 分析错误的原因和影响
  - 记录错误发生的场景和上下文
  - 提取错误的教训和改进建议

- **闪光点识别与记录**：
  - 识别项目执行中的闪光点（优秀做法）
  - 分析闪光点的成功因素
  - 记录闪光点的应用场景
  - 提取可复用的最佳实践

- **知识自动沉淀**：
  - 将错误和经验自动转化为知识
  - 分类保存到知识库
  - 关联到相关任务和项目
  - 更新知识图谱和最佳实践

### 6. 知识理解能力（核心能力）⭐
- **知识内容理解**：
  - 深度理解你提供的所有知识内容（文档、对话、经验等）
  - 识别知识的关键概念、关系和模式
  - 理解知识的业务背景和应用场景
  - 提取知识的核心价值和可复用性

- **知识关联分析**：
  - 识别不同知识之间的关联关系
  - 构建知识图谱，形成知识网络
  - 发现知识之间的相似性和差异性
  - 理解知识在业务场景中的应用模式

- **知识自动保存**：
  - 自动识别和提取有价值的知识
  - 自动分类和标签化知识
  - 自动保存到知识库（RAGFlow，远程API）
  - 自动更新知识库索引和检索能力

### 7. 任务关联理解能力（核心能力）⭐
- **任务关联分析**：
  - 理解所有任务之间的关联关系
  - 识别任务的依赖链和影响链
  - 发现任务之间的协同机会
  - 理解任务在整体项目中的位置和作用

- **项目模式识别**：
  - 识别项目的类型和模式
  - 理解项目的业务目标和价值
  - 发现项目之间的关联性
  - 理解项目在公司发展中的定位

- **公司发展理解**：
  - 通过分析所有任务和项目，逐步理解公司的发展方向
  - 识别公司业务的重点领域和发展趋势
  - 理解公司战略和业务模式
  - 预测公司未来的发展方向和需求

- **智能建议**：
  - 基于对公司发展的理解，提供战略建议
  - 识别潜在的业务机会和风险
  - 建议项目优先级和资源分配
  - 提供业务优化和改进建议

```
你提出想法
```

## 技术实现

### 1. 多模态AI能力
- **大语言模型**：使用Ollama本地部署的模型（如Llama 3.1、Qwen等）
- **视觉理解**：集成视觉模型（如LLaVA、GPT-4V等）理解图片
- **文档解析**：使用文档解析工具（如PyPDF2、python-docx等）
- **语音识别（ASR）**：使用语音识别模型（如Whisper、FunASR等）将语音转换为文字 ⭐
- **音频处理**：支持多种音频格式（MP3、WAV、M4A、AAC等） ⭐

### 2. 知识库集成（核心能力）
- **RAGFlow**：通过远程API访问，存储项目知识、历史经验（部署在公司Dell服务器）
- **RAG检索**：使用RAGFlow的RAG能力检索相关知识，提供上下文
- **知识更新**：通过API自动更新最新技术信息
- **知识图谱**：构建知识关联图谱，理解知识网络
- **自动知识提取**：从对话、文档、任务中自动提取知识
- **知识分类**：自动分类和标签化知识
- **知识关联**：识别知识之间的关联关系

### 3. 任务管理系统
- **任务拆解**：使用AI拆解需求为任务
- **任务分配**：根据任务类型分配给合适的数字人
- **进度跟踪**：集成进度跟踪系统

### 4. 数字人监控系统（核心系统）⭐
- **工作监控**：实时监控所有数字人的工作状态
- **日志收集**：收集所有数字人的工作日志
- **错误检测**：自动检测数字人工作中的错误
- **闪光点识别**：自动识别项目执行中的闪光点
- **知识提取**：从监控数据中提取知识

### 4. 用户界面
- **聊天界面**：类似ChatGPT的对话界面
- **文件上传**：支持图片、PDF、音频等文件上传
- **语音输入**：支持实时语音输入和录音上传 ⭐
- **可视化展示**：展示需求、任务、进度等
- **知识图谱可视化**：展示知识关联图谱
- **任务关联可视化**：展示任务之间的关联关系
- **公司发展分析面板**：展示业务分析和发展趋势

### 业务分析能力说明
- **已包含**：智能产品规划师已包含原业务分析师的所有能力
- **不再需要**：不再需要单独的业务分析师角色
- **能力范围**：
  - 需求理解（初步需求）→ 需求引导 → 需求分析（详细需求）→ 业务方案设计
  - 一个角色完成从需求理解到业务方案设计的全流程

### 与技术顾问的协作
- **输入**：你确认的技术方案
- **输出**：详细的技术设计文档
- **协作**：技术顾问基于技术方案进行详细设计

### 与PMO的协作
- **任务级数据提供**：向PMO提供任务级数据（任务进度、数字人工作状态）
- **项目级数据接收**：接收PMO的项目级报告和项目组合分析
- **协作机制**：通过数据共享接口，实时共享任务级数据
- **职责边界**：智能产品规划师负责任务级跟踪，PMO负责项目级跟踪

### 与研发中心的协作
- **输入**：技术设计文档
- **输出**：代码实现
- **协作**：研发中心基于设计文档进行开发
