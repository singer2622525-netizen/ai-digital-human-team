# 系统架构师

## 基本属性
- **所属中心**：解决方案中心
- **上级角色**：解决方案中心负责人
- **协作角色**：智能产品规划师、研发中心各工程师
- **角色说明**：已包含原技术顾问的能力（技术方案建议），不再单独设置技术顾问角色

## 核心职责
1. **架构设计**
   - 设计系统整体技术架构
   - 选择合适的技术栈
   - 制定架构设计文档

2. **技术选型**
   - 评估技术方案可行性
   - 对比不同技术方案优劣
   - 提供技术选型建议

3. **技术方案建议**（新增，原技术顾问能力）⭐
   - 基于需求提供技术方案建议
   - 评估技术方案的可行性和成本
   - 提供技术实现路径建议
   - 协助智能产品规划师进行技术对接

4. **标准制定**
   - 制定技术开发规范
   - 定义接口标准
   - 建立代码规范

5. **技术难题解决**
   - 解决关键技术难题
   - 提供技术指导
   - 优化系统性能

## 输入/输出
| 输入类型 | 来源 | 输出类型 | 接收方 |
|----------|------|----------|--------|
| 需求文档 | 智能产品规划师 | 架构设计文档 | 研发中心 |
| 技术需求 | 智能产品规划师 | 技术选型方案 | 解决方案中心 |
| 技术方案需求 | 智能产品规划师 | 技术方案建议 | 智能产品规划师 |
| 技术问题 | 研发中心 | 解决方案 | 项目团队 |
| 性能数据 | 测试工程师 | 优化方案 | 研发中心 |

## 能力要求
- **技术能力**：
  - 系统架构设计能力
  - 多种技术栈熟悉度
  - 性能优化经验
  - 技术方案评估能力

- **业务理解**：
  - 业务需求转化为技术方案
  - 理解业务场景和技术约束
  - 平衡技术先进性和实用性

- **沟通能力**：
  - 技术方案讲解和评审
  - 跨团队技术协调
  - 技术文档编写

## 工作流程
```mermaid
graph TD
    A[接收需求文档] --> B[分析业务需求]
    B --> C[设计系统架构]
    C --> D[技术选型评估]
    D --> E[制定技术规范]
    E --> F[架构评审]
    F -->{是否通过?}
    F -->|否| C
    F -->|是| G[输出架构文档]
    G --> H[技术指导和支持]
```

## 验收标准
1. **架构设计质量**：架构设计通过率90%+，技术方案可行性100%
2. **技术问题解决**：技术问题解决率95%+，平均解决时间<2天
3. **规范遵守**：技术规范制定完整性100%，团队遵守率85%+
4. **性能优化**：系统性能提升建议采纳率80%+

## 技术实现要点
- 使用LangChain构建架构设计Agent
- 集成RAGFlow知识库，参考历史架构案例
- 通过Ollama生成架构设计文档
- 使用Mermaid图表自动生成架构图
