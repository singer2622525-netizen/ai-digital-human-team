# RAGFlow 快速配置指南

## 🚀 三步完成配置

### 第一步：配置环境变量

编辑环境变量文件：

```bash
nano ~/DeveloperConfig/环境变量/.env.work
```

添加以下内容：

```bash
# RAGFlow配置
RAGFLOW_BASE_URL=https://ragflow.suuntoyun.com
RAGFLOW_USERNAME=你的RAGFlow用户名
RAGFLOW_PASSWORD=你的RAGFlow密码
```

**重要**：
- ✅ 使用环境变量存储密码（安全）
- ❌ 不要在代码中硬编码密码

### 第二步：设置文件权限

```bash
chmod 600 ~/DeveloperConfig/环境变量/.env.work
```

### 第三步：测试连接

```bash
cd "04-知识库建设/06-RAGFlow配置"
python3 test_ragflow_connection.py
```

---

## ✅ 如果测试通过

如果连接测试通过，就可以：

1. **创建知识库**
   ```bash
   python3 create_knowledge_bases.py
   ```

2. **导入初始知识**
   ```bash
   python3 import_initial_knowledge.py
   ```

## 📝 完整配置示例

```bash
# RAGFlow配置
RAGFLOW_BASE_URL=https://ragflow.suuntoyun.com
RAGFLOW_USERNAME=your-username
RAGFLOW_PASSWORD=your-password

# 知识库ID（创建后填写）
RAGFLOW_KB_ORGANIZATION=your-organization-kb-id
RAGFLOW_KB_PROJECT_EXPERIENCE=your-project-experience-kb-id
RAGFLOW_KB_TECHNICAL=your-technical-kb-id
RAGFLOW_KB_BUSINESS=your-business-kb-id
RAGFLOW_KB_DIGITAL_HUMAN=your-digital-human-kb-id
RAGFLOW_KB_ERROR=your-error-kb-id
RAGFLOW_KB_BEST_PRACTICES=your-best-practices-kb-id
RAGFLOW_KB_COMPANY_DEVELOPMENT=your-company-development-kb-id
```

---

## 🔒 安全提示

1. **不要提交密码到Git**
   - 确保 `.env.work` 在 `.gitignore` 中
   - 不要将密码写在代码中

2. **使用强密码**
   - 密码应该足够复杂
   - 定期更换密码

3. **保护环境变量文件**
   - 设置正确的文件权限（600）
   - 不要分享给他人
