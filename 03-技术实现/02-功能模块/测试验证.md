# 进度跟踪工具测试验证

## 测试步骤

### 1. 验证文件存在

在 Cursor 终端中运行：

```bash
# 检查核心文件
ls -la 03-技术实现/02-功能模块/progress_tracker.py
ls -la 03-技术实现/02-功能模块/init_progress.py
ls -la show_progress.sh
ls -la .vscode/tasks.json

# 应该看到所有文件都存在
```

### 2. 测试方式A：Cursor 任务（推荐）

1. 在 Cursor 中按 `Cmd+Shift+P` (Mac) 或 `Ctrl+Shift+P` (Windows)
2. 输入 "Run Task"
3. 选择 "📊 查看项目进度"
4. 应该看到进度仪表盘显示在终端面板

**预期结果**: 终端面板显示项目进度信息

### 3. 测试方式B：快速脚本

在 Cursor 终端中运行：

```bash
./show_progress.sh
```

**预期结果**: 显示进度仪表盘

### 4. 测试方式C：直接运行

在 Cursor 终端中运行：

```bash
python3 03-技术实现/02-功能模块/progress_tracker.py
```

**预期结果**: 显示进度仪表盘

### 5. 测试初始化（如果数据文件不存在）

在 Cursor 终端中运行：

```bash
python3 03-技术实现/02-功能模块/init_progress.py
```

**预期结果**:
- 如果数据文件不存在，会创建初始数据
- 如果数据文件已存在，会询问是否重新初始化

### 6. 测试添加任务

```bash
python3 03-技术实现/02-功能模块/progress_tracker.py add-task
```

按提示输入：
- 任务标题: 测试任务
- 关联里程碑ID: (直接回车跳过)
- 优先级: high
- 描述: (直接回车跳过)

**预期结果**: 显示 "✅ 已添加任务"

### 7. 测试查看所有任务

```bash
python3 03-技术实现/02-功能模块/progress_tracker.py list-tasks
```

**预期结果**: 列出所有任务，包括刚添加的测试任务

### 8. 测试更新任务状态

```bash
python3 03-技术实现/02-功能模块/progress_tracker.py update-task
```

按提示输入：
- 任务ID: t1 (或你看到的任务ID)
- 新状态: completed

**预期结果**: 显示 "✅ 任务状态已更新"

### 9. 再次查看进度

```bash
python3 03-技术实现/02-功能模块/progress_tracker.py
```

**预期结果**: 完成率应该有所提升

## 常见问题

### Q: 提示 "command not found: python3"
**A**: 检查 Python 是否安装：
```bash
which python3
python3 --version
```

### Q: 提示 "No such file or directory"
**A**: 确保在项目根目录运行命令，或使用完整路径

### Q: Cursor 任务不显示
**A**:
1. 检查 `.vscode/tasks.json` 文件是否存在
2. 重启 Cursor
3. 使用命令面板手动运行任务

### Q: 数据文件在哪里？
**A**: 数据文件保存在 `03-技术实现/02-功能模块/progress_data.json`
