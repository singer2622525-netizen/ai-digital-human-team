## 🚀 更新到GitHub的三种方式

### 方式1：使用更新脚本（最简单）⭐

```bash
# 方法A：交互式（会提示输入提交信息）
./更新到GitHub.sh

# 方法B：直接提供提交信息
./更新到GitHub.sh "更新说明：具体描述本次更新的内容"
```

### 方式2：标准Git命令

```bash
# 1. 查看更改
git status

# 2. 添加更改
git add .

# 3. 提交
git commit -m "更新说明：具体描述本次更新的内容"

# 4. 推送
git push
```

### 方式3：一键命令

```bash
git add . && git commit -m "更新说明" && git push
```

## 🔍 常用Git命令

### 查看状态
```bash
# 查看工作区状态
git status

# 查看具体更改内容
git diff

# 查看已暂存的更改
git diff --cached
```

### 提交管理
```bash
# 查看提交历史
git log --oneline -10

# 查看某个提交的详细信息
git show 提交ID

# 撤销最后一次提交（保留更改）
git reset --soft HEAD~1
```

### 远程仓库
```bash
# 查看远程仓库
git remote -v

# 拉取最新更改
git pull

# 查看远程分支
git branch -r
```

## 🆘 常见问题

### 问题1：推送失败 - 需要先拉取

**错误**: `Updates were rejected because the remote contains work...`

**解决**:
```bash
git pull
# 解决冲突后
git push
```

### 问题2：提交信息写错了

**解决**:
```bash
# 修改最后一次提交信息
git commit --amend -m "新的提交信息"
git push --force
```

### 问题3：想撤销某些文件的更改

**解决**:
```bash
# 撤销工作区的更改（未add）
git checkout -- 文件名

# 从暂存区移除（已add但未commit）
git reset HEAD 文件名
```
