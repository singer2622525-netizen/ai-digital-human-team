## 📋 后续工作更新流程

### 标准更新流程

```bash
# 1. 查看当前更改
git status

# 2. 查看具体更改内容
git diff

# 3. 添加更改的文件
git add .

# 或者选择性添加
git add 文件名或目录

# 4. 提交更改
git commit -m "更新说明：具体描述本次更新的内容"

# 5. 推送到GitHub
git push
```

### 更新提交信息格式建议

```
更新类型：更新内容简要说明

详细说明：
- 更新点1
- 更新点2
- 更新点3
```

**示例**：
